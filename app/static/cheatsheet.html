Pexo Cheat Sheet

Install it once. Use it as the local brain for your AI stack.

Pexo is a local-first buffer between you and Codex, Gemini, Claude, or other MCP-capable clients. It keeps memory, artifacts, preferences, sessions, and task state in one place on your machine.

Install Use the release bundle. It sets up Pexo, connects supported AI clients, runs diagnostics, and warms the runtime.
Use Start pexo, then let your AI client use Pexo as the shared local brain through MCP.
Maintain Use pexo --update, pexo doctor, and pexo connect all --scope user.

Install Pexo

The normal install path is the GitHub release bundle. It is the simplest and most stable option.

1

Download and run the bundle

Windows:

gh release download -R ParadoxGods/pexo-agent -p pexo-install-windows.zip --clobber
tar -xf .\pexo-install-windows.zip
.\install.cmd

macOS/Linux:

gh release download -R ParadoxGods/pexo-agent -p pexo-install-unix.tar.gz --clobber
tar -xzf pexo-install-unix.tar.gz
./install.sh
2

Verify the install

These commands should work immediately after setup.

pexo --version
pexo doctor --json
pexo connect all --scope user
3

Start Pexo

This opens the local control plane. From there, your connected AI clients can share one local memory and context layer.

pexo

Begin Using Pexo

The best way to use Pexo is not to chat with it directly. Use Codex, Gemini, or Claude normally and let them read and write through Pexo.

Normal flow

  • Start pexo.
  • Open your AI client.
  • Give it a normal task.
  • Let Pexo hold the memory, context, artifacts, and session state underneath.

Example prompt

Use Pexo as the shared local brain for this task.
Review this repo, tell me the top 3 concrete issues,
and store the result in Pexo memory.

Maintain Pexo

These are the only commands most users need for upkeep.

A

Update Pexo

Pull the latest managed release into your local install.

pexo --update
B

Run diagnostics

Check install health, runtime state, connected clients, and local paths.

pexo doctor
pexo doctor --json
C

Repair AI client connections

Reconnect Codex, Gemini, and Claude to the packaged MCP entrypoint if needed.

pexo connect all --scope user

Why Use Pexo

One local brain for multiple AI clients

Codex, Gemini, and Claude can share the same memory, artifacts, context, preferences, and task sessions.

Less repetition

You do not need to restate repo context, design preferences, or prior decisions every time you switch clients.

Local control

Pexo keeps state on your machine and gives you one place to inspect what the AI stack knows.

Better handoffs

One model can store context, another can continue the same work, and Pexo keeps the thread intact.

Repairable and observable

You can update it, diagnose it, reconnect clients, and inspect state without rebuilding your entire workflow.

What Pexo Is

Pexo is not meant to replace Codex, Gemini, or Claude. It is the stateful middle layer between you and them.

Think of it like this

User -> Pexo -> AI clients

Pexo handles memory, artifacts, preferences, sessions, and local control. The model still does the reasoning and generation.
