# Pexo Architecture & Agent Roles

## Core Orchestration Flow

1. **Intake:** Receive initial prompt from the user or connected model via the API.
2. **Clarification (The "One-Ask" Rule):** Immediately analyze the prompt and return *one* single follow-up question to the user to gather preferences, parameters, and constraints before any execution begins. This ensures all agents have the correct parameters.
3. **Extraction:** Break down the request into discrete tasks.
4. **Context Review:**
   - **Profiles:** Review user personality and scripting profiles from the database.
   - **Workspaces:** Review local project scopes and directory structures.
   - **Memory:** Retrieve contextual historical memory via vector search to keep token usage low.
5. **Execution:** Delegate the extracted tasks to the agent swarm.
6. **Persistence:** All agents autonomously write their state, logs, metrics, and final outputs back to the PostgreSQL database.

## The Agent Swarm

### Core Agents
*   **Supervisor Agent:** Oversees the entire operation, receives the extraction plan, and coordinates the delegation of tasks. It is the final reviewer before output is returned.
*   **Developer Agent:** Responsible for writing, editing, and reviewing code based on the Supervisor's instructions.

### Management Agents (The "Managers")
*   **Time Manager:** Monitors the duration of agent tasks, ensuring no agent hangs or exceeds allocated time limits. Can interrupt tasks that take too long.
*   **Context Cost Manager:** Tracks token usage and context window sizes across all agents, enforcing efficiency and preventing massive token bloat during memory/workspace retrieval.
*   **Resource Manager:** Monitors local system resources (CPU, RAM, Docker container stats) to ensure Pexo's local runs do not detrimentally impact the host machine's performance. It can throttle agent activity if system load gets too high.
*   **Code Organization Manager:** Enforces workspace structure, formatting standards, and optimal file placement for any code generated by the Developer Agent.
